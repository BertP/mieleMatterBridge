# Daily Development Log - 2026-04-21

## 📝 Zusammenfassung
Heute wurde der Grundstein für die visuelle Interaktion (OLED/QR) und die logische Verknüpfung zwischen der Miele Cloud API und dem Matter-Datenmodell gelegt. Wir haben den Übergang von einem reinen "Matter-Gerüst" zu einer funktionalen Bridge mit echtem Cloud-Mapping vollzogen.

## 🏗️ Architektur-Updates

### 1. UI-Subsystem (OLED & QR)
- **Komponente:** `ssd1306_simple` Erweiterung.
- **Änderung:** Implementierung einer skalierbaren QR-Code Rendering-Logik direkt auf dem ESP32-S3.
- **Zweck:** Headless-Onboarding. Die Bridge zeigt nun beim Start den Matter-Pairing-Code (QR) an, wenn sie noch nicht provisioniert ist.

### 2. Miele API Integration
- **Komponente:** `miele_client` & `http_server`.
- **Änderung:** Implementierung des OAuth Authorization Code Flows. 
- **Zweck:** Nutzer können die Bridge nun über ein lokales Web-Interface (`miele-bridge.local`) mit ihrem Miele-Konto verbinden, ohne Zugangsdaten im Code zu hinterlegen.

### 3. Matter Mapping Engine
- **Datei:** `main/miele_matter_mapping.cpp` & `.h`.
- **Änderung:** Erstellung einer dedizierten Mapping-Schicht, die JSON-Antworten der Miele-API in Matter-Cluster-Attribute übersetzt.
- **Unterstützte Felder:** `OperationalState`, `RemainingTime`, `DishwasherMode`.

## 🛠️ Implementierungs-Details

### OAuth Flow
- Der ESP32 startet einen Webserver auf Port 80.
- Redirect auf Miele Auth-Server -> Callback auf ESP32 -> Token-Austausch -> Speicherung im NVS.

### Matter Identity
- Festlegung einer stabilen Test-Identität für die Entwicklung:
  - **Passcode:** `12345678`
  - **Discriminator:** `1234`
  - **QR-Code:** Entsprechend generiert und im Code hinterlegt.

## 🔴 Offene Punkte & Risiken
- **Commissioning Timeout:** Wir beobachten teilweise Timeouts beim Pairing (Fail-safe timer expired). Vermutete Ursache: CPU-Last während der API-Abfragen.
- **NVS Persistence:** Validierung der Token-Wiederherstellung nach einem Hard-Reset steht noch aus.
- **Eventing:** Aktuell nutzen wir 60s Polling. Für eine "Wow"-Experience sollte auf SSE oder kürzere Intervalle gewechselt werden.
